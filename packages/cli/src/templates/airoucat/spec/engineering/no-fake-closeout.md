# No Fake Closeout

Do not turn partial work into a completion claim.

## Not Allowed

- Saying "done" because code was written.
- Calling unrun tests "verified".
- Treating CI as runtime evidence.
- Editing docs to say work is complete when code or evidence is missing.
- Hiding known failures behind default values, swallowed errors, or vague risk
  wording.

## Allowed

- "Implementation complete, verification not run."
- "Tests pass, runtime evidence missing."
- "Blocked by missing credential or environment."
- "PASS_WITH_RISK because the user accepted the recorded risk."
